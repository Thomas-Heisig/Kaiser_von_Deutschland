# Rollen-Perspektiven im Wirtschaftssystem

## Überblick

Jede Rolle im Spiel erlebt die Wirtschaft aus einer spezifischen, **begrenzten Perspektive**. Dies ist keine Vereinfachung, sondern die zentrale Spielmechanik: **Authentische Simulation bedeutet begrenzte Sicht**.

## Die Rollen-Perspektiven

### 1. Haushalt / Verbraucher

**Spielbare Rollen**: Bauer, Arbeiter, Handwerker (in privater Sphäre)

#### Wahrgenommener Wirtschaftsausschnitt

- **Sichtbar**:
  - Eigene Einnahmen (Lohn, Ernte)
  - Preise im lokalen Markt
  - Eigene Miete / Abgaben
  - Lokale Inflation ("Brot ist teurer geworden")
  - Knappheit ("Kein Fleisch verfügbar")

- **Unsichtbar**:
  - Makroökonomische Ursachen
  - Globale Handelsströme
  - Regierungspolitik (nur indirekt spürbar)
  - Gesamtwirtschaftliche Kennzahlen

#### Typische Gameplay-Mechanik

- **Ressourcenmanagement**: Begrenztes Budget optimal einsetzen
- **Budgetplanung**: Wofür gebe ich mein Geld aus?
- **Prioritäten setzen**: Essen vs. Kleidung vs. Wohnung
- **Notfallreserven**: Für schlechte Zeiten vorsorgen

#### Begrenzung der Perspektive

- **Makro-Mechanismen unsichtbar**: Warum sind die Preise gestiegen? Keine klare Antwort.
- **Entscheidungen wirken alternativlos**: "Ich MUSS diesen Job nehmen"
- **Keine Kontrolle über Rahmenbedingungen**: Steuern, Preise, Löhne sind extern vorgegeben
- **Informationsquellen**: Gerüchte, lokaler Markt, eigene Erfahrung

#### Herausforderungen

- Mit unvorhergesehenen Ereignissen umgehen (Preissprünge, Arbeitslosigkeit)
- Langfristige Planung trotz Unsicherheit
- Soziale Netzwerke nutzen (gegenseitige Hilfe)
- Aufstiegschancen erkennen und nutzen

#### Intensitäts-Profile

**Casual (Information: 3, Konsequenz: 3)**:
- Klare Budgetanzeige
- Preise relativ stabil
- Fehler sind korrigierbar

**Realistisch (Information: 8, Konsequenz: 9)**:
- Unerwartete Kosten
- Preise schwanken stark
- Verschuldung hat langfristige Folgen
- Arbeitslosigkeit führt zu Abwärtsspirale

---

### 2. Selbstständiger Handwerker / Händler

**Spielbare Rollen**: Handwerker, Händler, Gildenmeister

#### Wahrgenommener Wirtschaftsausschnitt

- **Sichtbar**:
  - Materialkosten (eigene Lieferanten)
  - Eigene Verkaufspreise
  - Kundennachfrage (direkt)
  - Konkurrenz (lokal sichtbar)
  - Steuern und Abgaben
  - Gilden-Regeln und Bürokratie

- **Unsichtbar**:
  - Gesamtwirtschaftliche Trends
  - Überregionale Handelsströme (außer wenn selbst beteiligt)
  - Regierungspolitik (bis auf eigenen Sektor)
  - Globale Ressourcenverfügbarkeit

#### Typische Gameplay-Mechanik

- **Preisgestaltung**: Was kann ich für mein Produkt verlangen?
- **Lagerverwaltung**: Wie viel Material auf Lager?
- **Kundenbeziehungen**: Stammkunden gewinnen und halten
- **Investitionen**: Werkstatt erweitern oder nicht?
- **Cashflow-Management**: Liquidität sicherstellen

#### Begrenzung der Perspektive

- **Makrotrends als unkontrollierbare "äußere Kräfte"**: "Warum kauft niemand mehr? Irgendwas ist los..."
- **Informationsasymmetrie zu Lieferanten**: Werden faire Preise bezahlt?
- **Regulierung als Hindernis oder Schutz**: Gildenregeln schränken ein, schützen aber auch
- **Begrenzte Skalierung**: Vom Handwerker zum Fabrikanten schwierig

#### Herausforderungen

- Konkurrenzdruck bewältigen
- Mit schwankender Nachfrage umgehen
- Bürokratie navigieren
- Geschäftserweiterung finanzieren
- Reputation aufbauen

#### Intensitäts-Profile

**Casual (Information: 4, Konsequenz: 4)**:
- Nachfrage relativ vorhersagbar
- Konkurrenzdruck moderat
- Insolvenz vermeidbar

**Realistisch (Information: 7, Konsequenz: 8)**:
- Nachfrage schwankt stark und unerwartet
- Materialpreise volatil
- Insolvenz bei Fehlentscheidungen
- Konkurrenz aggressiv
- Kreditaufnahme riskant

### 3. Minister / Ressortchef

**Spielbare Rollen**: Minister, Bürgermeister

#### Wahrgenommener Wirtschaftsausschnitt

- **Sichtbar**:
  - Ressort-spezifische Daten (Gesundheit, Bildung, etc.)
  - Budgetzuweisung für eigenes Ressort
  - Lobbygruppen und ihre Forderungen
  - Politische Mehrheiten
  - Zielkonflikte (Arbeit vs. Umwelt)

- **Teilweise sichtbar**:
  - Gesamtwirtschaftliche Lage (Berichte)
  - Auswirkungen eigener Politik (verzögert, teils unklar)
  - Internationale Entwicklungen (Berichte)

- **Unsichtbar**:
  - Detaillierte Mikro-Ebene (einzelne Haushalte)
  - Versteckte Marktdynamiken
  - Langfristige unbeabsichtigte Konsequenzen

#### Typische Gameplay-Mechanik

- **Makrosteuerung via Gesetze**: Mindestlohn, Umweltauflagen, Subventionen
- **Ressourcenallokation**: Budget verteilen
- **Interessenausgleich**: Verschiedene Gruppen zufriedenstellen
- **Regulierung**: Rahmenbedingungen setzen
- **Langfristplanung**: Infrastruktur, Bildung

#### Begrenzung der Perspektive

- **Maßnahmen wirken verzögert**: Bildungsreform zeigt Erfolg erst nach 20 Jahren
- **Unbeabsichtigte Konsequenzen**: Mindestlohn → Schwarzarbeit
- **Keine Vollkontrolle**: Bürger und Unternehmen reagieren nicht wie gewünscht
- **Informationsasymmetrie nach oben**: Berichte sind gefiltert, beschönigt
- **Politischer Druck**: Schnelle Erfolge vs. langfristig richtige Entscheidungen

#### Herausforderungen

- Zielkonflikte managen (Wirtschaftswachstum vs. Umweltschutz)
- Mit begrenzten Ressourcen priorisieren
- Lobbygruppen widerstehen
- Langfristige Vision gegen Kurzfrist-Druck durchsetzen
- Komplexe Systeme verstehen (Bildung, Gesundheit)

#### Intensitäts-Profile

**Casual (Information: 5, Konsequenz: 5)**:
- Berichte sind relativ akkurat
- Politikmaßnahmen wirken schneller
- Korrekturmaßnahmen möglich

**Realistisch (Information: 7, Konsequenz: 9)**:
- Berichte widersprüchlich, verzögert
- Maßnahmen brauchen Jahre bis Jahrzehnte
- Pfadabhängigkeit stark
- Lobbying und Korruption beeinflussen Entscheidungen
- Unbeabsichtigte Konsequenzen häufig

### 4. Wissenschaftler / Analytiker

**Spielbare Rollen**: Gelehrter, spezialisierte Berater

#### Wahrgenommener Wirtschaftsausschnitt

- **Sichtbar**:
  - Daten, Statistiken, historische Trends
  - Korrelationen zwischen Variablen
  - Modelle und Theorien
  - Forschungsergebnisse (eigene und anderer)

- **Teilweise sichtbar**:
  - Ursache-Wirkungs-Zusammenhänge (Hypothesen)
  - Zukünftige Entwicklungen (Prognosen)

- **Unsichtbar**:
  - Vollständiges Kausalitätsnetz
  - Versteckte Variablen
  - Zufällige Ereignisse
  - Politische Machtspiele

#### Typische Gameplay-Mechanik

- **Datenanalyse**: Muster in Statistiken erkennen
- **Hypothesenbildung**: Warum passiert X?
- **Modellierung**: Simulationen erstellen
- **Prognosen**: Zukunft vorhersagen
- **Beratung**: Empfehlungen für politische Entscheider

#### Begrenzung der Perspektive

- **Kaum direkte Eingriffsmöglichkeit**: Kann nur beraten, nicht entscheiden
- **Modelle sind immer unvollständig**: Vereinfachungen notwendig
- **Unsicherheit bei Kausalität**: Korrelation ≠ Kausalität
- **Theorien konkurrieren**: Verschiedene Schulen widersprechen sich
- **Black-Swan-Events**: Unvorhersehbare Schocks

#### Herausforderungen

- Komplexe Systeme verstehen
- Relevante von irrelevanten Daten trennen
- Modelle validieren
- Erkenntnisse kommunizieren
- Mit Unsicherheit umgehen
- Politische Entscheider überzeugen (die oft nicht zuhören)

#### Intensitäts-Profile

**Casual (Information: 6, Interpretation: 4)**:
- Daten relativ vollständig
- Modelle funktionieren gut
- Empfehlungen werden eher befolgt

**Realistisch (Information: 9, Interpretation: 9)**:
- Daten lückenhaft, widersprüchlich
- Modelle versagen oft
- Empfehlungen werden ignoriert
- Rivalisierende Theorien
- Unerwartete Entwicklungen häufig

### 5. "Gott"-Perspektive (Debug/Lern-Modus)

**Zugriff**: Nur über spezielles Menü aktivierbar

#### Wahrgenommener Wirtschaftsausschnitt

- **Vollständig sichtbar**:
  - Alle Variablen im System
  - Alle Beziehungen zwischen Variablen
  - Versteckte Mechanismen
  - Zukünftige geplante Events
  - KI-Entscheidungslogik

#### Typische Gameplay-Mechanik

- **Reine Beobachtung**: Zusehen ohne Eingriff
- **Parameter-Experimente**: "Was wäre wenn...?"
- **Debugging**: Warum passiert X?
- **Lernen**: Verstehen der Systemmechaniken

#### Zweck

**Keine reale Rolle**, sondern:
- Kontrastmittel zu anderen Perspektiven
- Zeigt die Begrenztheit normaler Rollen
- Lern-Tool für Spieler
- Debug-Tool für Entwickler
- Pädagogisches Werkzeug für Lehrer

#### Pädagogischer Wert

- **Vergleich**: Gott-Perspektive vs. Minister-Perspektive zeigt Informationsasymmetrie
- **Einsicht**: "Aha, DESHALB ist die Wirtschaft kollabiert!"
- **Verständnis**: Komplexität wird sichtbar
- **Demut**: Realisierung, wie schwer Steuerung wirklich ist

## Perspektiven-Wechsel als Kernmechanik

### Lernzyklus

1. **Spielen als Haushalt**: Erleben von Preisänderungen, Arbeitslosigkeit
2. **Wechsel zu Minister**: Versuchen, Probleme zu lösen
3. **Wechsel zu Gott-Perspektive**: Verstehen, warum es so schwer war
4. **Zurück zu Haushalt**: Neue Wertschätzung für Komplexität

### Multiplayer-Dimension

- **Verschiedene Spieler, verschiedene Rollen**: Jeder sieht die Wirtschaft anders
- **Konflikte aus Perspektiven**: Minister glaubt, alles läuft gut; Arbeiter leiden
- **Verhandlungen**: Basierend auf unterschiedlichen Informationen
- **Emergenz**: Globale Muster aus lokalen Perspektiven

## Technische Umsetzung

### Informationsfilter

```typescript
interface RolePerspective {
  visibleData: string[]; // Welche Daten sind sichtbar?
  dataAccuracy: number; // 0.0-1.0, wie genau sind die Daten?
  dataDelay: number; // In Tagen, wie alt sind die Informationen?
  interpretationHelp: boolean; // Werden Zusammenhänge erklärt?
}

const perspectives: Record<RoleType, RolePerspective> = {
  HOUSEHOLD: {
    visibleData: ['ownIncome', 'localPrices', 'localAvailability'],
    dataAccuracy: 0.95,
    dataDelay: 0,
    interpretationHelp: false
  },
  CRAFTSMAN: {
    visibleData: ['ownIncome', 'materialCosts', 'customerDemand', 'localCompetition'],
    dataAccuracy: 0.9,
    dataDelay: 1,
    interpretationHelp: false
  },
  MINISTER: {
    visibleData: ['macroStats', 'budgets', 'policies', 'lobbyGroups'],
    dataAccuracy: 0.7,
    dataDelay: 7,
    interpretationHelp: true
  },
  SCIENTIST: {
    visibleData: ['allStats', 'historicalData', 'correlations'],
    dataAccuracy: 0.8,
    dataDelay: 14,
    interpretationHelp: false
  },
  GOD: {
    visibleData: ['*'], // Alles
    dataAccuracy: 1.0,
    dataDelay: 0,
    interpretationHelp: true
  }
};
```

### UI-Anpassung

Jede Rolle hat eigene UI-Layouts:
- **Haushalt**: Einfaches Budget-Dashboard
- **Händler**: Warenlisten, Preistabellen
- **Minister**: Politikmenü, Statistik-Übersichten
- **Wissenschaftler**: Graphen, Korrelationsmatrizen
- **Gott**: Vollständiges Debug-Panel
